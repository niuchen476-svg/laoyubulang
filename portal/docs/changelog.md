## 如何记录新条目

```markdown
### YYYY-MM-DD · 功能名称 · 一句话描述

!!! note "类型"
    简短说明这次变更的背景。

- 变更点 1
- 变更点 2
```

!!! tip "约定"
    - `success` = 里程碑 / 重大功能上线
    - `note` = 常规更新
    - `warning` = 破坏性变更 / 注意事项
    - `bug` = 缺陷修复
